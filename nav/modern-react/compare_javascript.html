8 Aug 2025 - 자바스크립트의 동등 비교

1. 개인적으로 javascript 가 내 주언어인 java 와 다른점만 기억하기위해 정리해보았다.

  • undefined 는 선언됐지만 값이 할당되지 않은것. null 은 명시적으로 비어있다 라고 표현할때 사용
  • if(literal) 에서 literal 이 falsy 한 값일 경우 동작한다. java 는 boolean 만 조건문으로 동작하는데 javascript 는 falsy 한 값만으로도 동작한다. 
    • false, (0,-0,0n,0x0n), NaN, ('',"",``), null, undefined
  • 'number' 는 -(Math.pow(2,53) -1) ~ Math.pow(2,53) 까지만 저장한다. Bigint 는 더 큰 숫자를 저장 가능하다. 사용법은 숫자n or Bigint(숫자)
  • ₩ (백틱) 을 쓰면 줄바꿈이 표현되고 표현식을 쓸 수 있다.
  • 문자열은 원시타입이며 변경이 불가능하다. (java 에서는 문자열은 특수하게 객체처럼 다뤄지는데)
    💡 책 코드 예제
    
const foo = 'bar';
foo[0] = 'a';
console.log(foo); // 결과 = bar (바뀌지않음)
    💡 다양한 시도
    
const foo = 'bar';
foo = 'aaa';
console.log(foo); // 결과 = bar (바뀌지않음)

let foo = 'bar';
foo[0] ='a';
console.log(foo); // 결과 = bar (바뀌지않음)

foo = 'aaa'
console.log(foo); // 결과 = aaa 바뀜
    문자열이 불변이기때문에 문자 하나만 바꾸려는 시도는 불가능하고, 문자열 자체를 교체하는것은 const, let 에 대해서만 적용되는것
